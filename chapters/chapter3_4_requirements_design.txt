  TECHNOLOGY JUSTIFICATION:

  - PYTHON: Chosen for its simplicity, extensive library ecosystem, and
    strong support for AI/ML and computer vision applications.

  - OPENCV: The most widely-used open-source computer vision library,
    providing efficient real-time video processing capabilities.

  - FACE_RECOGNITION LIBRARY: Built on dlib's deep learning model, it
    provides simple API calls for face detection and recognition with
    99.38% accuracy on standard benchmarks.

    dashboards without the overhead of larger frameworks like Django.

  - SQLITE: A serverless, zero-configuration database that stores data
    in a single file - perfect for a standalone application.

  - BOOTSTRAP: Enables rapid development of responsive, professional-
    looking user interfaces.

4.5 MODULE DESCRIPTION
-----------------------

  MODULE 1: FACE REGISTRATION MODULE
  -----------------------------------
  Purpose: Allows admin to register new users and train the recognition model

  Sub-module A - Photo Capture (Live Camera):
    - Admin enters user name and email, then clicks "Open Camera"
    - Browser requests webcam access via getUserMedia API (requires
      localhost or HTTPS; blocked on plain HTTP + IP by browser policy)
    - Live <video> element streams camera feed in real-time
    - "Start Capture" triggers automatic capture of 50 frames at 350ms
      intervals using a <canvas> element drawing from the live video
    - Each frame is JPEG-encoded client-side and collected as base64
    - On completion, all frames are sent as JSON to /register_camera
    - Server saves frames to static/registered_faces/<name>/ and adds
      the user to the database with trained=0 (untrained state)
    - No face encoding is computed at this stage

  Sub-module B - Upload Photo (fallback):
    - Admin uploads a single JPG/PNG photo
    - System extracts face encoding from the photo immediately
    - User is registered with trained=1 (immediately active)

  Sub-module C - Train Model:
    - Admin clicks "Train Model" in the dedicated panel on the
      registration page
    - System queries all users with trained=0
    - For each untrained user, all saved photos are read from disk
    - Face encodings are extracted from each photo and averaged into
      a single robust 128-dimensional vector
    - The averaged encoding is appended to the engine's known_encodings
    - The model is saved to face_encodings.pkl
    - The user's trained column is updated to 1 in the database
    - The admin sees a summary: trained users and any failures

  MODULE 2: FACE DETECTION MODULE
  --------------------------------
  Purpose: Detects human faces in real-time video feed
  Functions:
    - Capture video frames from webcam/IP camera
    - Apply HOG or CNN-based face detection
    - Locate face positions (bounding boxes)
    - Draw rectangles around detected faces
    - Pass detected face regions to recognition module

  MODULE 3: FACE RECOGNITION MODULE
  -----------------------------------
  Purpose: Identifies detected faces by comparing with database
  Functions:
    - Extract face encoding from detected face
    - Compare encoding with all stored encodings
    - Calculate Euclidean distance for similarity
    - Identify the person if distance < threshold (0.6)
    - Flag as "Unknown" if no match found

  MODULE 4: ATTENDANCE MANAGEMENT MODULE
  ----------------------------------------
  Purpose: Manages attendance records automatically
  Functions:
    - Log attendance when a recognized face is detected
    - Record check-in time with timestamp
    - Prevent duplicate entries for same day
    - Calculate attendance statistics
    - Mark late arrivals based on configured time

  MODULE 5: WEB DASHBOARD MODULE
  --------------------------------
  Purpose: Provides web-based interface for system interaction
  Functions:
    - Admin login and authentication
    - Live camera feed display
    - User management interface
    - Attendance viewing with date filters
    - Report generation and download
    - System settings and configuration

  MODULE 6: ALERT & NOTIFICATION MODULE
  ----------------------------------------
  Purpose: Handles security alerts for unknown faces
  Functions:
    - Detect unrecognized/unknown faces
    - Capture and store unknown face images
    - Log unknown face events with timestamp
    - Display alerts on dashboard
    - Maintain alert history

  MODULE 7: REPORT GENERATION MODULE
  ------------------------------------
  Purpose: Generates attendance reports for analysis
  Functions:
    - Generate daily/weekly/monthly attendance reports
    - Export reports in CSV format
    - Filter reports by date range or user
    - Calculate attendance percentage
    - Summarize attendance statistics
